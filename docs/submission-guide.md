# Submission Guide

Submit your system's predictions to the TCGC leaderboard via a GitHub Pull Request.

## Step-by-step

### 1. Fork the repository

Fork `sargonxg/TCGC_TACITUS-Conflict-Grammar-Corpus_BENCHMARK` and clone your fork.

### 2. Run your system over the sample items

```bash
pip install -e '.[dev]'
# Run your system — output must be a JSONL file where each line has:
# { "id": "tcgc-NNNN", ...gold-shaped fields... }
```

### 3. Score locally

```bash
tcgc score your_predictions.jsonl items/v0.1-sample/ --out scores.json
tcgc report scores.json
```

### 4. Create your submission directory

```
submissions/
  <your-system-name>/
    <YYYY-MM-DD>/
      predictions.jsonl     # your model outputs
      SYSTEM_CARD.md        # generated by: tcgc card scores.json
      submission.json       # metadata (see schema below)
```

### 5. Fill in `submission.json`

```json
{
  "system_name": "My System v1.0",
  "system_version": "1.0.0",
  "contact_email": "you@example.com",
  "predictions_path": "submissions/my-system/2026-04-18/predictions.jsonl",
  "system_card_path": "submissions/my-system/2026-04-18/SYSTEM_CARD.md",
  "scorer_commit": "abc1234"
}
```

Get the scorer commit hash: `git rev-parse --short HEAD`

### 6. Open a Pull Request

Push your branch and open a PR against `main`. The leaderboard CI will:

1. Validate `submission.json` against `schema/submission-v0.1.json`
2. Re-run `tcgc score` on your predictions
3. Post the per-task-type results as a PR comment

### 7. Merge and leaderboard update

Once the PR is merged, results appear in `leaderboard/v0.1-sample.json`.

---

## Alternatively: email submission

Send `predictions.jsonl` and `SYSTEM_CARD.md` to `hello@tacitus.me`. Include system name, version, and contact.
